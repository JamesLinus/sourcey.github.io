HTML5 Video is Finally Ready

Today I had a pleasant surprise when I was able to get a H.264 MPEG-4 AVC encoded MP4 video to play on Chrome (34.0.1847.131), Firefox (28.0) and Safari (5.1.7) using the native HTML5 video element. Cross-browser HTML5 video (for the important browsers that is) is finally a go, keep on reading to find out how.

As expected, Safari plays the video when source is provided with the video/mp4 mime type. Here's the trick though, both Chrome and Firefox will play the same file if it's provided with a video/webm attribute. Yes, this is a hack and MP4 files shouldn't be served with video/webm, but the point is that it works.

Here is the proof of concept. Try it out in Chrome, Firefox and Safari:

Please note that MP4 doesn't work out-of-the box on Firefox because of software patents, you need to have a thrid-party decoder installed. See [this page](https://developer.mozilla.org/en-US/docs/HTML/Supported_media_formats) on MDN for more information.

And the source code:

<video controls="controls">
    <source src="http://media.sourcey.com/big_buck_bunny.mp4" type='video/mp4'/>
    <source src="http://media.sourcey.com/big_buck_bunny.mp4" type='video/webm'/>
</video>

This is exactly what us developers wanted from HTML5 video since the beginning; a ubiquitous video format that works across all major browsers! No more having to store multiple encodings of each damn video just to cater for different browser vendors and their ridiculous licensing strategies. Well, now we have it, and it seems that H.264 has won the race.

Unfortunately the prevailing video codec is about as patent encumbered as a NASA spaceship, which begs the next question; what's left for WebM? At this stage WebM's "openness" may be its only saving grace.

I also had some luck getting this method to work with videos hosted Google Drive. Test results and working examples are posted here: HTML5 Video Streaming from Google Drive.

If you're able to test this on the new Internet Explorer I'd love to know if it works or not. Drop me a comment if you're able to test it, or have any other thoughts to add.
